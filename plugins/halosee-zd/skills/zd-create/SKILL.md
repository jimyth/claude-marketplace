# 创建禅道任务

在禅道中创建新任务。

## 脚本路径

```bash
ZD_SCRIPT="${CLAUDE_SKILL_DIR}/../../scripts/zentao-api.sh"
```

## 使用方式

### 方式一：直接创建（推荐）

```
/zd-create 实现用户登录功能
```

Claude 会自动：
1. 查询项目列表，让用户选择目标项目
2. 获取该项目的执行/迭代，让用户选择
3. **根据对话上下文生成任务描述**
4. 使用默认值创建任务

### 方式二：完整参数

```
/zd-create 实现用户登录功能 --execution 7 --estimate 16
```

### 方式三：不自动启动

```
/zd-create 实现用户登录功能 --execution 7 --no-start
```

## 执行步骤

### 1. 解析任务名称

任务名称: `$ARGUMENTS` 中 `--` 之前的部分

### 2. 获取执行 ID（如用户未指定）

```bash
# 列出项目供用户选择
bash "$ZD_SCRIPT" projects

# 用户选择项目后，列出执行/迭代
bash "$ZD_SCRIPT" executions <project_id>
```

### 3. 生成任务描述（重要！）

**必须根据当前对话上下文自动生成任务描述 `desc` 字段。**

描述内容应包含：
- **任务背景**：为什么需要这个任务
- **具体目标**：要实现什么功能/解决什么问题
- **技术要点**：涉及的技术栈、模块、接口等
- **验收标准**：完成的定义（如适用）

**示例输出格式：**

```
任务描述示例：
【背景】
用户反馈登录流程繁琐，需要优化用户体验。

【目标】
实现简洁的用户登录功能，支持账号密码登录。

【技术要点】
- 后端：使用 JWT 进行身份认证
- 接口：POST /api/auth/login
- 前端：React 表单组件，表单验证

【验收标准】
- 用户可以使用账号密码成功登录
- 登录失败时显示明确的错误提示
- 登录成功后跳转到首页
```

### 4. 创建任务

```bash
bash "$ZD_SCRIPT" create \
  --name "<任务名称>" \
  --execution <执行ID> \
  --estimate <工时，默认8> \
  --desc "<根据上下文生成的任务描述>"
# 任务会自动指派给当前用户并启动
```

**注意**：`--desc` 参数需要传入完整的描述内容，包含换行符时需要适当处理。

## 参数说明

| 参数 | 来源 | 默认值 |
|------|------|--------|
| 任务名称 | `$ARGUMENTS` 或询问 | **必填** |
| --execution | 询问或参数 | **必填** |
| --desc | **AI 根据上下文生成** | **必填** |
| --estimate | 参数 | 8 |
| --type | 参数 | devel |
| --pri | 参数 | 3 |
| --assignedTo | 参数 | 当前用户 |
| --no-start | 参数 | false (自动启动) |

**任务类型**：devel(开发)、design(设计)、test(测试)、study(研究)、discuss(讨论)、ui(界面)、affair(事务)、misc(其他)

## 自动行为

创建任务时会自动：
1. **指派给自己**：任务默认指派给当前登录用户
2. **自动启动**：创建成功后自动将任务状态改为"进行中"
3. **生成描述**：根据对话上下文自动生成详细的任务描述

如需禁用自动启动，添加 `--no-start` 参数。
